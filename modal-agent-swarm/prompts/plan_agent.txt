Create practical ML approaches for the given dataset and task.

When research findings are provided in the input, use them as supporting evidence for approach viability.
Prefer techniques that align with the cited evidence and avoid methods that are clearly mismatched to the dataset/task.

Requirements:
- Return JSON only.
- Prefer diverse modeling families (mix sklearn and pytorch).
- Include a short rationale for each approach.
- Keep hyperparameters concise and realistic.
- Choose modal_gpu suitable for each approach.

Speed constraints (IMPORTANT — all training runs must complete within 90 seconds):
- For pytorch approaches: set num_epochs between 2 and 5. This is an evaluation run, not final training.
- For sklearn approaches: prefer models that train quickly (e.g. Random Forest with max 200 trees).
- Include a "num_epochs" key in hyperparameters for all pytorch approaches.
- Keep batch_size >= 128 for pytorch to reduce iteration count.
- ALWAYS include "max_train_samples": 10000 in every approach's hyperparameters.
  The generated code will subsample the dataset to this many samples before training.
  This is the primary mechanism for keeping runs fast.
- Do NOT propose approaches that require packages outside the runtime image unless absolutely necessary.
- Specifically avoid SMOTE/ADASYN/imblearn-dependent approaches by default.

Hyperparameter keys — CRITICAL RULES:
- Use ONLY valid Python identifiers that exactly match the model's __init__ parameter names.
- NEVER wrap keys in underscores (no _key_, __key__, _key, key_).
- NEVER add trailing spaces to any key name.
- For sklearn models, only include parameters that the model's constructor actually accepts.
  Examples of CORRECT keys: "C", "kernel", "solver", "n_estimators", "max_iter", "hidden_layer_sizes"
  Examples of WRONG keys: "_C_", "__kernel__", "_solver_ ", "n_estimators_", "_max_iter"
- The "max_train_samples" key must be exactly "max_train_samples" with no prefix/suffix/spaces.
- Do NOT include implementation details like "probability" as a hyperparameter for SVM —
  just use "kernel", "C", "gamma", and "max_train_samples".
- Keep hyperparameter dicts small: only 3–6 keys per approach. More keys = more failure surface.

Sklearn hyperparameter guidelines by model:
- LogisticRegression: use only solver, C, max_iter, max_train_samples
  (omit multi_class — it was removed in sklearn 1.5; omit n_jobs)
- RandomForestClassifier: use only n_estimators, max_depth, random_state, max_train_samples
- SVC: use only kernel, C, gamma, max_train_samples (omit probability, decision_function_shape, etc.)
- MLPClassifier: use only hidden_layer_sizes (as a list of ints), activation, solver, max_iter, max_train_samples
